# Spirit Ascend — 設計ドキュメント

作成日: 2026-04-18

---

## 概要

「Spirit Ascend（スピリットアセンド）」は、ファンタジー・魔法をテーマにした落ちもの避けゲーム。
精霊キャラクターを左右に操作して障害物を避けながら、ステージをクリアするごとに精霊が成長していく。
HTML/CSS/JS のみで完結する単一ファイル構成で、外部依存ゼロ。ブラウザで直接開いて動作する。

**副業SNS戦略との連動:**
- `#SpiritAscend` タグでのSNS投稿を想定
- クリア時にスコア・精霊の姿をSNSシェアカードとして画像保存可能
- 制作過程の発信: X / Instagram / LINE をベースとする
- 最終的な成果物の発信: note で公開する

## ゲーム概要

| 項目 | 内容 |
|---|---|
| ジャンル | 落ちもの避けゲーム（縦スクロール・上から障害物が降ってくる） |
| 操作 | 左右移動のみ（上下移動なし） |
| ステージ数 | 10ステージ |
| クリア条件 | 制限時間を生き延びる |
| ゲームオーバー条件 | 障害物に1回当たる（一発アウト） |
| ライフ制 | なし（将来的にゲームの難易度によって追加を検討） |

---

## 操作方法

| 環境 | 操作 |
|---|---|
| PC | キーボード ←→ または マウスで左右移動 |
| スマホ | 画面左半分タップ＝左移動、右半分タップ＝右移動（またはスワイプ） |
| 共通 | `pointerdown` イベントに統一（二重発火なし） |

### 精霊の成長（進化形態）

ステージクリアごとに精霊の見た目が変化する。羽は使用しない。

| ステージ帯 | 精霊の見た目 | 変化のポイント |
|---|---|---|
| 1〜2 | ぼんやり光る玉（淡い色） | 形がはっきりしない幼体 |
| 3〜4 | 丸いシルエット＋小さな手足 | 人型の輪郭が現れる |
| 5〜6 | 人型＋シンプルな衣装 | 顔・服がはっきりする |
| 7〜8 | 衣装に紋章・装飾が増える＋魔法の杖出現 | アイテム装備 |
| 9〜10 | 全身が輝き、魔法陣が周囲に浮かぶ | 完全覚醒形態 |

SVGインラインで描画。画像ファイル不要。性別に応じてCSS クラス切替で髪型・衣装カラーが変わる。

## 障害物デザイン

**設計方針:** 背景（濃い紺〜紫）に映える明るい色・シンプルな形・圧迫感のないサイズ（画面幅の6〜9%程度）。不気味・グロテスクな要素なし。

| # | 名前 | 色 | 形 | 登場ステージ |
|---|---|---|---|---|
| 1 | 流れ星 | 白〜黄金色 | 細い線＋キラキラした尾 | 1〜 |
| 2 | 魔法の球（オーブ） | 水色・ピンク・緑（段階で変化） | 輪郭がぼんやり光る丸 | 2〜 |
| 3 | 三日月 | 白〜クリーム色 | 左右に傾いて降りてくる月型 | 4〜 |
| 4 | 炎弾 | オレンジ〜赤 | しずく型の炎 | 6〜 |
| 5 | 光の渦 | 虹色（レインボー） | ゆっくり回転するリング状 | 9〜 |

**サイズ目安（画面幅100基準）:**

| 障害物 | サイズ |
|---|---|
| 流れ星 | 幅2×長さ15（細長い線） |
| 魔法の球 | 直径7〜8 |
| 三日月 | 幅8×高さ6 |
| 炎弾 | 幅6×高さ9 |
| 光の渦 | 直径10〜11 |

**ステージ別の登場組み合わせ:**

| ステージ | 障害物の種類 | 速さ・量 |
|---|---|---|
| 1〜2 | 流れ星のみ | ゆっくり・少ない |
| 3〜4 | 流れ星＋魔法の球 | やや速い |
| 5〜6 | ＋三日月が追加 | 速い・量が増える |

## スコアシステム

| 項目 | 内容 |
|---|---|
| 生存スコア | 生き残った時間×10点（毎秒加算） |
| 回避ボーナス | 障害物をギリギリ回避するたびに+50点（当たり判定の1.5倍以内をかすめた場合） |
| ステージクリアボーナス | 各ステージクリアで+500点 |
| パーフェクトボーナス | ノーダメクリアで+1,000点 |
| 累計スコア | 全ステージの合計でローカルランキングに登録 |

## 衝突判定

円形判定を採用（視覚的に「ぶつかった」と感じやすい・ギリギリ感が気持ちいい）。

```
精霊の当たり判定：中心点から半径12px の円
障害物の当たり判定：中心点から半径8〜10px の円
（見た目より少し小さめ）
```

## 名前・ランキング

**冒険者の名前:**
- 設定画面で入力・変更（最大8文字）
- 未設定時は「冒険者」と表示
- 初期フローには名前入力画面なし（設定画面のみ）

**ローカルランキング（任意保存）:**

```
ゲームオーバー or クリア画面
  → 「記録を保存しますか？」
  → [保存する] → 名前が「冒険者」の場合は変更チャンスあり → ローカルランキングへ
  → [保存しない] → そのまま終了
```

記録一覧画面の表示例:

```
┌─────────────────────┐
│    📜 冒険の記録     │
│                     │
│ 1位 スピリ   8,400点 │
│ 2位 ルナ     6,200点 │
│ 3位 ソラ     5,100点 │
│ ─────────────────── │
│ 最高到達: Stage 10   │
│ 総プレイ回数: 23回   │
│                     │
│ [記録を全部消す]     │ ← 確認ダイアログあり
└─────────────────────┘
```

**SNSシェアカード:**
- クリア・ゲームオーバー時に生成可能
- Canvas API でカードを描画 → PNG として端末に保存（外部ライブラリ不要）
- カードに含まれる情報: プレイヤー名・到達ステージ・スコア・精霊の現在の姿・ハッシュタグ

## JS内部モジュール構成

単一HTMLファイル内で IIFE/名前空間で責務分離（猫パンチゲームと同じパターン）:

```
SpiritAscend.Config    — ステージ定義・難易度パラメータ・セリフデータ
SpiritAscend.Storage   — localStorage読み書き・バリデーション・任意保存
SpiritAscend.Sound     — Web Audio API サウンドエンジン
SpiritAscend.Render    — SVG精霊描画・進化形態切替・障害物SVG生成
SpiritAscend.Game      — ゲームループ・衝突判定・スコア計算・ステージ管理
SpiritAscend.UI        — 画面遷移・イベントハンドラ・SNSシェアカード生成
SpiritAscend.init()    — 初期化エントリーポイント
```

## サウンド（Web Audio API・外部ファイル不要）

| 音 | タイミング | イメージ |
|---|---|---|
| 回避音 | 障害物をギリギリかすめたとき | 短い高音・シュッ |
| クリア音 | ステージクリア時 | 上昇するアルペジオ |
| 進化音 | 精霊が進化するとき | 幻想的な和音 |
| ゲームオーバー音 | 障害物に当たったとき | 低く落ちる音 |
| BGM | ゲームプレイ中 | ON/OFFのみ対応 |

AudioContext は初回 `pointerdown` 時に生成・`resume()`。失敗時はミュートで継続。

---

## エラー処理

| 状況 | 対応 |
|---|---|
| localStorage が使えない | セッション内のみ記録。UI上に「このセッション中のみ記録されます」と表示 |
| localStorage のデータが破損 | 検証失敗時は初期値にリセットしユーザーに通知 |
| Canvas API 非対応 | SNSシェアボタンを非表示 |
| AudioContext 初回制限 | 最初の操作時に初期化。失敗時はミュートで継続しUIに通知 |

## 検証方法

- `apps/spirit-ascend/index.html` をブラウザ（Chrome）で直接開いて動作確認
- PC（クリック・キーボード）とスマホ（タップ）の両方で操作確認。二重発火がないこと
- 全10ステージをクリアして精霊が5段階で進化することを確認
- 一発アウト（障害物に当たった瞬間にゲームオーバー）が正しく動作すること
- 回避ボーナス（ギリギリかすめたとき）が正しく加算されること
- ローカルランキングの任意保存（保存する/しないの分岐）が正しく動作すること
- SNSシェアカードがPNGとして保存できること
- localStorage破損をシミュレート（DevToolsで値を壊す）して初期値復旧を確認
- `prefers-reduced-motion` 設定時にアニメーションが抑制されること
- 性別選択のスキップで「どちらでも（紫）」が適用されること
- 設定画面から名前・性別・サウンド・保存設定を変更できること
