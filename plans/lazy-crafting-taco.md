# 怒れる猫パンチマシン — 実装プラン

## Context

落ち込んだ時に元気を出せるWebアプリを作成したい。API課金なし・外部依存なしで、HTML/CSS/JavaScriptのみで完結する「怒れる猫パンチマシン」を実装する。ストレス発散（連打パンチ）と癒し（ゴロゴロ演出）を組み合わせた、ユニークでユーモアのあるアプリ。

## 対応環境

- **対応ブラウザ**: Chrome, Edge, Firefox, Safari（最新版）
- **起動方式**: `file://` で直接開く（主要ブラウザでlocalStorageが動作する）。ただしlocalStorageが使用不可の場合はセッション内のみデータ保持し、UI上で通知する
- **レスポンシブ**: スマホ（タッチ）/ PC（クリック）両対応

## ファイル構成

```
apps/neko-punch/
  index.html    ← 全コード内包（HTML + CSS + JS）の単一ファイル
```

ブラウザで直接開くだけで起動可能。外部依存ゼロ。

## JS内部構造（単一ファイル内のモジュール分割）

単一HTMLファイル内で、JSをIIFE/名前空間で責務分離する:

```
NekoPunch.Sound    — Web Audio API サウンドエンジン
NekoPunch.Render   — SVG描画・表情切り替え・アニメーション
NekoPunch.Game     — コンボ・スコア・称号・実績ロジック
NekoPunch.Storage  — localStorage永続化（バリデーション付き）
NekoPunch.UI       — イベントハンドラ・DOM操作
NekoPunch.init()   — 初期化エントリーポイント
```

## 主要機能

### 1. 猫キャラクター（インラインSVG）
- 画像ファイル不要。SVGで猫を描画（体・頭・耳・目・鼻・口・ヒゲ・前足・しっぽ）
- 4つの表情状態を切り替え:
  - **待機（怒り顔）**: 細め目 + V字眉 + への字口
  - **パンチ中**: 見開き目 + 開口
  - **コンボ中**: 炎目 + ニヤリ口 + 赤オーラ
  - **ゴロゴロ（甘え）**: 細い線目 + ω口 + ハートエフェクト

### 2. パンチアクション
- **入力イベント**: `pointerdown` に統一（click/touchの二重発火を回避）
- 猫が左右交互にパンチ（200msアニメーション）
- クリック地点に「バシッ!」テキストがポップアップして消える
- `touch-action: manipulation` でダブルタップズーム無効化

### 3. コンボシステム
- 最後の操作から1.5秒でリセット
- コンボ段階に応じて演出がエスカレート:

| コンボ | 演出 |
|--------|------|
| 1-4 | 通常パンチ |
| 5-9 | 猫の目が光る + 画面軽く揺れ |
| 10-19 | 炎エフェクト + 背景が赤みがかる |
| 20-29 | 猫が巨大化 + 画面シェイク |
| 30+ | 猫パンチ無双: 回転 + レインボー背景 + パーティクル |

### 4. ゴロゴロ演出（コンボ10以上でリセット時に発動）
1. 怒り顔 → 幸せ顔にトランジション
2. 猫の体が微振動（ゴロゴロ）
3. ハートが5-8個浮かび上がる
4. 甘えセリフ表示
5. 3秒後に怒り顔に戻る（「べ、別に甘えてないし!」）

### 5. 猫の一言（辛辣だけど愛がある）
- 通常セリフ20個、コンボ中セリフ5個、ゴロゴロセリフ7個を内蔵
- 例: 「おまえ、今日も生きてるだけで偉いぞ。…まあ当然だけどな」
- お気に入り保存機能付き（ハートボタン）

### 6. サウンド（Web Audio API — 外部ファイル不要）
- **パンチ音**: ホワイトノイズバースト + 低音サイン波
- **コンボ音**: 半音ずつ上昇するピッチ（12コンボでオクターブ上がってリセット）
- **ゴロゴロ音**: 25-30Hzの低周波サイン波を断続的に再生
- **実績解除音**: 上昇アルペジオ（ド-ミ-ソ-高ド）
- 設定でON/OFF可能
- **AudioContext初期化**: 初回 `pointerdown` 時に生成し `resume()` を呼ぶ。失敗時はミュート状態で継続し、UIに「音声が再生できません」と通知する

### 7. ゲーミフィケーション
- **スコア**: 1パンチ = 10点 + コンボボーナス（コンボ数×5）
- **称号**: 累計パンチ数で昇格（猫パンチ見習い → 猫パンチ神 → 猫そのもの）
- **実績**: 10種類（初パンチ、30コンボ、深夜プレイ、秒速10パンチなど）

### 8. データ保存（localStorage）
- 累計パンチ数、ハイスコア、最大コンボ、称号、実績、お気に入りセリフ、連続プレイ日数
- **バリデーション**: 読み込み時にスキーマ検証・型チェック・範囲クランプを実施。破損・改ざんデータは初期値にフォールバック
- **localStorage使用不可時**: セッション内のみデータ保持。UI上に「データ保存が利用できません」と表示

### 9. レスポンシブ対応
- モバイルファースト設計（縦3分割: スコア / 猫 / セリフ）
- PC大画面では最大幅480pxのカード風表示
- `touch-action: manipulation` でダブルタップズーム無効化
- `prefers-reduced-motion` で激しいアニメーション抑制（シェイク・パーティクルを無効化）

## 実装ステップ

1. HTMLスケルトン + CSSレイアウト + レスポンシブ基盤
2. SVG猫キャラクター描画 + 4つの表情定義
3. `pointerdown` イベント処理 + 基本アニメーション（左右交互パンチ）
4. コンボシステム + 段階的演出（シェイク、炎、レインボー）
5. Web Audio API サウンドエンジン（初回操作時にAudioContext初期化）
6. ゴロゴロ演出（表情変化 + ハート + ゴロゴロ音）
7. セリフ表示システム + 全セリフデータ埋め込み
8. スコア・称号・実績システム
9. ローカルストレージ永続化（バリデーション付き）
10. パーティクル・エフェクト最終調整

## 検証方法

- `apps/neko-punch/index.html` をブラウザ（Chrome）で直接開いて動作確認
- PC（クリック）とスマホ（タップ）の両方で操作確認。二重発火がないこと
- コンボ10以上→1.5秒放置でゴロゴロ演出が発動するか確認（境界値: 9コンボでは発動しない）
- コンボ30+でレインボー演出が発動し、フレーム落ちが体感上問題ないこと
- サウンドON/OFF切り替えの確認。初回操作前は音が鳴らないこと（AudioContext仕様）
- ブラウザリロード後にデータが保持されているか確認
- `prefers-reduced-motion` 設定時にシェイク・パーティクルが無効化されること
- localStorage破損をシミュレート（DevToolsで値を壊す）して初期値復旧を確認
